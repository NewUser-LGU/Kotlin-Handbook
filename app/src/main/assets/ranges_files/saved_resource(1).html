        Спасибо :)
        Добавлю, что диапазоны (ranges) целочисленных типов Int, Long, Char представляют прогрессии (progressions): IntProgression , LongProgression, CharProgression. Прогрессии характеризуются наличием свойств: first - первый элемент, last - последний элемент, step - шаг, и могут быть отсортированы в цикле for.
        Т.к., прогрессия реализует интерфейс Iterable<T> , где T - Int , Long, Char, ее можно использовать в функциях сбора и фильтрации: map , filter и т.д.

                    Последовательности в памяти храниться не будут, они видны только компилятору.
                    Код на Kotlin

                    for (c in 0..9)
                    print (c)

                    будет аналогичен коду на Java

                    for (c = 0; c <= 9; c++)
                    print (c)

                    Оба примера скомпилируются в один и тот же байт-код.

                    Код на Kotlin

                    var range = 0..99
                    var isInRange = 15 in range
                    println (isInRange)

                    будет аналогичен коду на Java

                    println(15 > 0 && 15 <= 99)

                    Код

                    var range = 0..99
                    var isInRange = 15 in range
                    println (!isInRange)

                    будет, соответственно,

                    println (!(15 > 0 && 15 <= 99))
