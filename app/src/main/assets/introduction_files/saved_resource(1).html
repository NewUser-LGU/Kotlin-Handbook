                            Denis • год назад • edited

                            Большой минус, автор не написал про консольный ввод в Kotlin. Для ввода строки используется функция readLine(), результатом будет String? (null при достижении конца файла, при вводе с клавиатуры не встречается). Пример:

                            fun main() {
                                print("Input your name: ")
                                val name = readLine()
                                if (name != null) {
                                    println("Hello $name!")
                                }
                            }


                            Также Kotlin Native не упомянут. На нем просто вызываем компилятор без параметров: <каталог с Kotlin Native>\bin\kotlinc app.kt - в Windows будет создан файл program.exe. Для указания имени исполняемого файла вызывать kotlinc -o App.exe app.kt
                            P. S. Более продвинутым программистам советую изучить исходники Kotlin Native на гитхабе. Отличный пример, как создать свой кроссплатформенный компилятор (Windows, Linux, MacOS) с помощью LLVM. Код хорошо читаем и не очень большой.

Kotlin это упрощенная, улучшенная Scala для масс, примерно как Java - это упрощенный C++ для энтерпрайза.
                                                          Ну и в целом довольно таки хороший язык - сравните, например, с убогим Ceylon, придуманным в RedHat. Есть практически все функциональные фишки из Scala. Компиляция в JVM (энтерпрайз), JavaScript (Web), а теперь и в native код. Если они сделают еще и полноценную компиляцию под iOS (для полной замены Swift), то у Kotlin есть все шансы стать языком №1 для мобилок.
                                                          Единственное, что не понравилось в синтаксисе - слово fun (def как-то привычнее, но это вкусовщина) и обязательные скобки в условиях, вроде if (<cond>) { ...} или while (<cond>) { ... }. Лишний ; убрали, можно было бы и скобки убрать, сделать if <cond> { ... } (а фигурные скобки - обязательными, даже для одного оператора). Меньше лишнего в синтаксисе, более выразительный код.
